- 若快速排序在平均情况下的运行是境为O(n * log n), 而合并排序的的运行时间总是 O(n * log n), 为何不使用合并排序？ 它不是更快吗？
    - 算法实际运行时间为 c*O，一般情况下不考虑常量c, 但有时常量影响很大，对于快速排序和合并排序就是如此。快速排序的常量比合并排序小，因此如果它们的复杂度都为O(n * log n), 快速排序的速度将更快。实际中，也更快。因为实际中遇上平均情况的可能性要比最坏情况大得多。
